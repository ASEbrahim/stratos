## 7. Key Changes from V2 to V2.2

| Dimension | V2 | V2.2 |
|-----------|----|----|
| Base model | Qwen3-8B | **Qwen3.5-9B** |
| Teacher model | Claude Opus (distillation) | **Gemini 3 Flash** |
| Training examples | ~5,000 (est.) | **11,785** |
| Eval examples | ~500 (est.) | **1,296** |
| Number of profiles | ~10 | **~45** |
| Scoring method | Opus re-scoring existing scans | **Fresh Gemini scoring of 25 profiles x 2K articles** |

### What Worked
1. **Gemini Flash as teacher** — cheaper than Opus, high quality scores, allowed 2.3x more training data
2. **Qwen3.5-9B base** — larger model capacity without exceeding 24GB VRAM for DoRA training
3. **WeightedRandomSampler** — upsampled rare high-relevance articles by 4x, preventing noise dominance
4. **Per-sample loss weighting** — critical articles weighted 6x more than noise in loss computation
5. **No think_text in training** — avoided the V2.1 catastrophe (MAE=3.244 from CoT in training data)

### What Didn't Work (Flagged Profiles)
- **HR director, Construction PM, Veterinarian** — likely insufficient diversity in training articles for these domains
- **Backend software engineer** — only 2 eval examples (insufficient data, not a model failure)
