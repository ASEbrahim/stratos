## 2. Infrastructure Tests

### 2.1 VRAM Coexistence

**Test:** Can scorer model (8.7GB) + wizard model (9.3GB) coexist in 24GB VRAM?

| Configuration | Loaded Size | Result |
|---------------|:-----------:|:------:|
| Scorer only | ~9.3GB (model + KV cache) | OK |
| Scorer + qwen3:14b | ~18.6GB + KV cache overhead | **FAILED** — exceeds 24GB |
| Scorer + qwen3.5:35b-a3b | ~32GB | Impossible |

**Result:** Model swapping is unavoidable on 24GB GPU. Keep `OLLAMA_MAX_LOADED_MODELS=1`.

### 2.2 Parallel Inference (OLLAMA_NUM_PARALLEL)

| Setting | Throughput | Relative |
|---------|:----------:|:--------:|
| NUM_PARALLEL=1 | 1.00x (baseline) | 100% |
| NUM_PARALLEL=4 | 0.94x | **94% (6% slower)** |

**Result:** GPU serializes all inference requests. Parallelism adds scheduling overhead without throughput benefit. Keep `NUM_PARALLEL=1`.

## 4. Code Changes & Bug Fixes

### 4.1 Prompt Template Version-Pinning (`prompt_version.py`) — NEW

Prevents the #1 failure mode from V2/V2.1: prompt format drift between training export and inference.

- Hashes the prompt template skeleton (strips dynamic values like role, article content)
- Registers hashes from both `export_training.py` and `scorer_adaptive.py`
- Logs `WARNING` if training and inference templates produce different hashes
- Strips runtime-only additions (LANGUAGE line, feedback blocks) before hashing

**Commit:** `329fd66`

### 4.2 Tracked Fields Join Bug Fix (`scorer_adaptive.py`)

**Bug:** `', '.join("Equate, SLB")` was iterating characters, producing `"E, q, u, a, t, e, ,,  , S, L, B"`.

**Fix:** Type-check before joining — only call `join()` on lists, pass strings through directly.

**Impact:** No production impact yet (current profiles store tracked_companies as strings), but would have broken any profile using tracked fields as lists (e.g., from wizard/profile generator).

**Commit:** `372e358`
