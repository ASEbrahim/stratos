# StratOS — Session Handoff (March 14, 2026)

## What StratOS Is

StratOS is a personalized strategic intelligence dashboard. It fetches news from multiple sources, scores them using a local Ollama-hosted LLM, and presents a prioritized feed. It has 6 AI personas (Intelligence, Market, Scholarly, Gaming, Anime, TCG), a YouTube knowledge extraction pipeline, a file browser, scenario/RPG system, and a self-improving scoring model via Claude Opus distillation.

### 4. DB Lock Contention Fix (CRITICAL)
- **Root cause**: YouTube worker thread held `strat.db.conn` per-thread connection during minutes-long Whisper transcription, preventing WAL checkpointing and blocking ALL other writers

- All YouTube mutating endpoints (delete, cancel, retranscribe, reprocess) also use independent connections
- `database.py _commit()` has Python lock — DO NOT REMOVE IT (removing it was tried and caused cascade failures)
- Stuck videos reset to `'failed'` on server restart (NOT `'pending'` — that re-queues them and locks the DB)

## Architecture — Critical Knowledge

### Database Threading Model
```
strat.db = Database() (singleton)
  └─ self._local = threading.local()  # Per-thread connections
  └─ self.lock = threading.Lock()      # Python lock for _commit()
  └─ conn property creates connection per thread (lazy, never closed)

RULE: Daemon threads (YouTube worker, retranscribe threads) MUST NOT use strat.db.conn
      They must use independent sqlite3.connect(str(strat.db.db_path)) with busy_timeout
      Otherwise they hold read locks that prevent WAL checkpointing → cascade DB lock
```

### YouTube Pipeline Architecture
```
Video Lifecycle: pending → transcribing → transcribed/low_quality/failed → extracting → complete

Transcript Acquisition (3-tier):
  Tier 1: youtube-transcript-api (api.list() + track.fetch()) — manual captions preferred
  Tier 2: Supadata API (paid, if key configured)
  Tier 3: faster-whisper (yt-dlp downloads audio → whisper transcribes on CPU)

Video Discovery: YouTube RSS feed → yt-dlp --flat-playlist fallback
Worker: youtube_worker.py daemon thread, 30s startup delay, processes 1 video at a time
Lens Extraction: On-demand via POST /api/youtube/extract-lens (7 lenses)

CJK Detection: _detect_cjk_language() in youtube.py
  Hiragana/Katakana → ja, Hangul → ko, CJK ideographs only → zh

yt-dlp: use --js-runtimes node (NOT nodejs, NOT deno)
```

### Auth Token Pattern
```javascript
// CORRECT — use everywhere in frontend:
const token = typeof getAuthToken === 'function' ? getAuthToken() : '';
headers: { 'X-Auth-Token': token }

// WRONG — these keys were fixed but watch for regressions:
// localStorage.getItem('auth_token')           — WRONG
// localStorage.getItem('stratos_session_token') — WRONG
// localStorage.getItem('stratos_token')         — WRONG
// Authorization: Bearer ...                     — WRONG (backend only reads X-Auth-Token)
```

### Route Dispatch
```
server.py do_GET/do_POST/do_DELETE calls handle_get/handle_post/handle_delete
on each route module sequentially. First to return True wins.

Route modules (16): agent, agent_tools, auth, config, controls, data_endpoints,
dev_endpoints, feeds, generate, helpers, media, persona_data, persona_prompts,
personas, wizard, youtube_endpoints
```

### UI Settings Sync
```
ui-sync.js patches localStorage.setItem/removeItem → debounced POST to /api/ui-state
~65+ keys synced. Theme overrides via stratos-theme-custom-{theme} keys.
_uiSyncSuppressed flag prevents feedback loop during load.
beforeunload flushes via navigator.sendBeacon.
```
