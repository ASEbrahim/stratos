### Fullscreen Agent — Bug Fixes (Iterative)

**Problem 1: Fullscreen showed nothing (only executive summary)**
- Root cause: `#agent-panel` is nested inside `#feed-column` inside `#main-content`. Setting `opacity: 0` on `#main-content` hid the agent panel too (opacity creates a stacking context that traps children).
- Fix: Agent panel now reparents to `<body>` when entering fullscreen, restores to original DOM position on exit. Tracks original parent/sibling references via `panel._origParent` and `panel._origNext`.

**Problem 2: Minimizing fullscreen lost the entire agent panel**
- Root cause: Exit code used `wrapper.querySelector('.rounded-xl')` to find the chat content, but the sidebar's "New Chat" button also has `rounded-xl` class — it matched the wrong element.
- Fix: Tags the real chat content with `data-agent-inner="1"` on enter, queries `[data-agent-inner]` on exit. Fallback logic moves all non-sidebar children if attribute is missing.

**Problem 3: Executive summary visible behind fullscreen**
- Root cause: CSS was hiding `#main-content` but the executive summary header is inside `<main>` (parent of `#main-content`), not `#main-content` itself.
- Fix: CSS now targets `body:has(.agent-fullscreen) > .flex > main` with `visibility: hidden !important; opacity: 0 !important`.

**Problem 4: Theme/particles not showing in fullscreen**
- Root cause: `.agent-fullscreen` had `background: var(--bg-app)` — a solid opaque background covering the star canvas.
- Fix: Background changed to `transparent`. Chat area and sidebar use `rgba(8,8,26,0.7)` with `backdrop-filter: blur(16-20px)` — frosted glass effect lets theme particles show through.

**Problem 5: Context/Files panels not interactable in fullscreen**
- Root cause: Both panels were appended to `document.body` with `position: absolute` and no z-index. Agent fullscreen has `z-index: 9999`, so they rendered behind it.
- Fix: Both panels now use `position: fixed; inset: 0; z-index: 10000`.

**Problem 6: Context and Files panels overlapping**
- Root cause: Opening one didn't close the other — both could be visible simultaneously.
- Fix: `toggleContextEditor()` closes file browser if open; `toggleFileBrowser()` closes context editor if open.

**Problem 7: Sidebar opacity/blur customizer not working**
- Root cause: CSS had `background: rgba(...) !important` on `.agent-fs-sidebar` which overrode the JS inline style. Also, the sidebar HTML template had an inline `background` style.
- Fix: Removed `!important` from CSS background rule. JS now uses `sidebar.style.setProperty('background', ..., 'important')`. Removed inline background from sidebar HTML template.

**Problem 8: Typography/border radius customizer not working**
- Root cause: CSS had hardcoded `font-size: 15px` and `!important` rules that overrode inline styles set by the customizer. Also, per-element inline styles only affect existing bubbles, not new messages.
- Fix: All fullscreen typography now uses CSS custom properties (`--fs-font-size`, `--fs-line-height`, `--fs-bubble-padding`, `--fs-bubble-radius`, `--fs-input-height`, `--fs-send-size`). Customizer sets these on the panel element, so they apply to all current and future elements via CSS cascade.

**Problem 9: Actions not updating sidebar in real time**
- Root cause: 6 key functions (`_deleteConversation`, `_switchConversation`, `newAgentChat`, `_saveAgentHistory`, `switchPersona`, `clearAgentChat`) called `_renderConvTabs()` (compact view) but never checked `_agentFullscreen` to rebuild the fullscreen sidebar.
- Fix: Added `if (_agentFullscreen) _refreshFsSidebar()` after `_renderConvTabs()` in all 6 functions.

**Problem 10: New Chat button still not updating in real time (after fix 9)**
- Root cause: Race condition. Sidebar's inline `onclick="newAgentChat();_refreshFsSidebar()"` fired `_refreshFsSidebar()` synchronously before the async `newAgentChat()` completed. The sidebar rebuilt with stale data. Also, `_refreshFsSidebar` itself re-fetched from the server (another async race).
- Fix: (a) Removed all duplicate `_refreshFsSidebar()` from inline onclick handlers — functions handle it internally after `await`. (b) `_refreshFsSidebar()` no longer re-fetches; it rebuilds from the already-loaded `_agentConvList`.

**Problem 11: UI Scale slider too aggressive + glitching**
- Root cause: Range was 0.7-1.5 with step 0.05 (too coarse, too wide). Also, `zoom` was applied to the entire `.agent-fs-wrapper` which includes the customizer panel itself — the panel zoomed while you dragged its own slider.
- Fix: (a) Range narrowed to 0.85-1.3 with step 0.01. (b) Zoom now applies to `.agent-fs-sidebar` and `[data-agent-inner]` individually, excluding the customizer panel.
