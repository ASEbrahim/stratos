## Architecture

```
News Sources → NewsFetcher → AI Scorer → SQLite DB → JSON API → Frontend
     ↑              ↑            ↑                         ↓
  RSS/DDG/     ThreadPool    Qwen3.5-9B              Vanilla JS SPA
  Serper/CF     Executor      DoRA V2.2              (61K+ lines)
                                 ↑                         ↓
                          Feedback Loop              User Feedback
                                 ↑                         ↓
                     LoRA Training ← Export ← Distillation (Gemini/Opus)
```

### Key Modules
- **Backend**: main.py (orchestrator), server.py (HTTP), auth.py, database.py, sse.py
- **Routes**: agent.py, generate.py, wizard.py, config.py, helpers.py
- **Processors**: scorer_adaptive.py (sole scorer), scorer_base.py, briefing.py, profile_generator.py
- **Fetchers**: news.py, market.py, discovery.py, extra_feeds.py, serper_search.py, kuwait_scrapers.py
- **Frontend**: 14 JS modules, index.html, styles.css (8 themes × 3 modes = 24 variants)

## Platform Vision

Four modes sharing the same scoring engine:
1. **Intelligence Mode** (current) — Bloomberg Terminal-like for professionals
2. **Anime/Manga Mode** — AniList-inspired (60% UI already built)
3. **Collection/TCG Mode** — TCGPlayer-inspired (70% infra reuse via TradingView)
4. **Gaming Mode** — Steam-inspired (50% infra reuse)

- Profile isolation not fully scoped (SSE, is_url_seen)
- Chat history not persisting
